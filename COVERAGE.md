# Corpus Coverage

This document describes the scope and completeness of the regulatory data included in this MCP server.

## Data Source

**Malta Financial Services Authority (MFSA)**
- URL: https://www.mfsa.mt/
- Role: Single regulator for financial services in Malta

## Sourcebooks

| Sourcebook ID | Name | Description |
|---------------|------|-------------|
| `MFSA_Rules` | MFSA Rules | Binding rules issued under the Financial Services Act and sector legislation |
| `MFSA_Guidance_Notes` | MFSA Guidance Notes | Interpretive guidance on regulatory expectations |
| `MFSA_Circulars` | MFSA Circulars | Official circulars to industry on regulatory developments |

## Coverage Scope

| Category | Status |
|----------|--------|
| Investment Services rules | Included |
| AML/CFT obligations | Included |
| Corporate governance requirements | Included |
| Enforcement actions (administrative penalties) | Included |
| Enforcement actions (licence withdrawals) | Included |
| Enforcement actions (public warnings) | Included |
| Banking sector rules | Partial — see note below |
| Insurance sector rules | Partial — see note below |
| Virtual Financial Assets (VFA) rules | Partial — see note below |

> **Note:** Coverage of sector-specific rules depends on the completeness of the last ingest run. The ingest script (`scripts/ingest-mfsa.ts`) crawls the MFSA publications index. Run `npm run ingest` to refresh.

## Data Age

The corpus data age is tracked via the `MFSA_DATA_AGE` environment variable. The default is `2024-01-01` for the sample database.

Use `mt_fin_check_data_freshness` to query the data age at runtime.

## Known Gaps

- Historical versions of provisions (pre-effective-date text) are not retained; only current/latest text is stored.
- MFSA consultation papers and discussion documents are not ingested.
- Cross-border passporting notifications are out of scope.

## Refresh Process

1. Run `npm run ingest` (or trigger `ingest.yml` workflow)
2. Review ingested data for accuracy
3. Update `MFSA_DATA_AGE` environment variable
4. Rebuild Docker image via `ghcr-build.yml`
